TranslateResponse 枚举重构
Plain | Dict + DictEntry

把翻译结果从单译文 struct 重构为可判别枚举，为词典源（TV4-F2/F3）腾出结构化承载，且既有 19 源（机翻 + LLM）零行为回归。前后端类型对齐 serde tag，Plain 渲染不回归，Dict 渲染留 TV4-F4。

类型　coding · 大功能 commit　PENDING 更新　2026-06-06 author　coder
一

做了什么

把翻译结果类型从单译文 struct 重构为可判别枚举 TranslateResponse，为后续词典源腾出结构化承载，且既有源零行为回归。

  • 后端 TranslateResponse 改为 internally tagged 枚举（serde tag = "kind"、rename_all = "lowercase"）：Plain { translated } → {"kind":"plain","translated":"..."}；Dict { entry } → {"kind":"dict","entry":{...}}。
  • 新增 DictEntry（音标 / 按词性分组释义 PosDefinition / 例句 / 发音 / 词形变化，全 Option/Vec 容空）与便捷构造 TranslateResponse::plain(...)。
  • 全部既有源 parse_response 改返回 Plain 变体（构造点统一走 TranslateResponse::plain）。
  • IPC 边界 TranslateResultDto 升级带判别标签：新增 kind（"plain"/"dict"）+ 可选 entry；translate_text_impl 按 variant 拆分映射，Dict 经 dict_entry_summary 压成纯文本写历史并回退展示。
  • 前端 TranslateResult 改为可判别联合（TranslatePlainResult | TranslateDictResult），新增 TS DictEntry/PosDefinition；消费处补 kind: "plain"。Dict 渲染组件留 TV4-F4，本步只保证类型编译通过、Plain 渲染不回归。
二

关键决策与理由

D1

enum 用 struct variant

配合 serde internally tagged，要求各变体序列化为 JSON object；tuple/newtype 无法内联 tag，struct variant 把 kind 与字段同层平铺，前端拿到扁平 {kind, translated} 即可判别。

D2

tag = "kind" + lowercase

判别字段名 kind 与前端可判别联合习惯一致；lowercase 让 tag 值为 "plain"/"dict"，前后端字符串字面量对齐、避免大小写错配。

D3

DTO 保留扁平三字段 + 加 kind/entry

既有前端组件与测试大量依赖 result.translated/sourceLang/targetLang，Plain 路径字段不变 → 渲染与历史写入零回归；Dict 路径 translated 填摘要兜底，老组件有可读回退。

D4

dict_entry_summary 兜底

历史表 translated_text 是纯文本列，词条无法整存；取首批释义按词性扁平化 join 成单行，无释义再退音标，保证历史栏可读、不丢信息。

D5

entry 用 skip_serializing_if

Plain 结果不带 entry 键，前端联合 plain 分支无该字段，序列化产物与 TS 类型严格对齐。

D6

集成测试各自加 plain_text helper

tests/providers.rs 与 tests/translate.rs 是独立 crate，无法复用单测模块 helper，各自加一份从 Plain 取译文、遇 Dict panic 的小工具。

四

坑与注意

  • 同名字段误伤：.translated 在 cache.rs（CacheEntry DB 列）与 tests/translate.rs 的 CacheEntry { translated } 处同名但无关；批量 perl 替换只针对 TranslateResponse 变量（ok/ok_str/single/multi/resp），未误伤；line 2049 字符串字面量亦保留。
  • partial-move：matches! 模式绑定按值 move，失败消息里再 {resp:?} 会报错 → 统一用 matches!(&resp, ...) 借用。
  • RTK 折叠输出：cargo test 输出被折叠成 "N passed" 摘要、丢 per-test 行；验证三冻结命中与 test result: ok 时用 rtk proxy cargo test 取原始输出存 artifacts。
