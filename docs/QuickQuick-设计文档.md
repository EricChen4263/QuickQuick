# QuickQuick 设计文档

> 跨平台（macOS + Windows）托盘常驻小工具，自用、直接分发、无 App Sandbox。两大核心功能：**剪贴板历史** + **翻译**。
> 本文档是产品方向与架构约束的权威依据，沉淀自逐节点拷问（grill）形成的设计共识，供 feature-dev 七阶段开发参照。

- **适用范围**：QuickQuick v1（Phase 0–3 全功能）+ v2 跨机同步设计
- **技术栈**：Tauri 2.0（Rust 后端）+ React 前端；存储 SQLCipher（加密 SQLite）
- **制定**：2026-05-31 · **状态**：设计冻结，待进入实现

---

## 一、产品定位与范围

QuickQuick 是一个跨平台桌面小工具，托盘常驻 + 全局热键弹窗，主窗口仅管设置和全量历史。定位**自用、直接分发、无 App Sandbox**。仓库起步为空（无历史包袱）。

| 维度 | 定论 |
|---|---|
| 平台 | macOS + Windows |
| 分发 | 直接分发（无商店沙盒约束） |
| 核心功能 | 剪贴板历史 + 翻译 |
| 形态 | 托盘常驻 + 全局热键弹窗；主窗口管设置与全量历史 |
| v1 范围 | Phase 0 骨架 / 1 文本剪贴板 / 2 翻译核心 / 3 图片+富文本+收藏+筛选+App排除+加密+改键UI |
| v2 范围 | 跨机同步（数据层 v1 即预埋 UUID+时间戳+可变状态时间戳+墓碑） |

**默认热键**：历史 = `Cmd/Ctrl+Shift+C`、翻译 = `Cmd/Ctrl+Shift+T`，**必须可改键**。

## 二、技术栈与系统形态

- **后端**：Tauri 2.0（Rust）。全局热键 mac 走 Carbon `RegisterEventHotKey`（无需授权），Win 走系统热键 API。
- **前端**：React（webview）。
- **存储**：嵌入式 SQLCipher，引擎静态编入 App，数据即单文件 `quickquick.db`，首次启动自动创建。位置：mac `~/Library/Application Support/QuickQuick/`、Win `%APPDATA%\QuickQuick\`。每台机器各一份、互相独立（v1 不同步）。
- **自启动**：Tauri autostart 插件，onboarding 提供开关、默认开。
- **自动更新**：v1 即上 Tauri updater 插件（配套签名密钥 + 托管更新清单与安装包，如 GitHub Releases）。

### 权限与首次运行

| 平台 | 关键授权 | 说明 |
|---|---|---|
| macOS | **Accessibility** | 模拟粘贴（Cmd+V）与读取选中文字必须授权；全局热键不需要 |
| Windows | 无 | 剪贴板 / SendInput / 全局热键均无需特殊授权 |

**Accessibility 引导 = 主动引导 + 优雅降级**：检测 `AXIsProcessTrusted` → 未授权弹说明卡片 + 一键深链跳系统设置›辅助功能 → 轮询自动继续；未授权时功能降级不崩（自动粘贴退化为仅写回剪贴板）。

**热键冲突 = 注册时检测 + 即时提示**：绑定失败当场提示"已被占用"并拒绝保存，改键实时校验，出厂默认值已验证。

## 三、剪贴板捕获 / 回写 / 隐私机制

这是剪贴板工具的技术心脏——所有"操纵剪贴板"的动作（捕获、回写粘贴、选中即译读选中）共用一套机制，边角最多、最易出竞态。

| # | 分支 | 定论 | 否决项 |
|---|---|---|---|
| 1 | 选中文字获取 | 模拟 Cmd+C + 读后恢复原剪贴板 + 打标记防自捕（AX API 作 fast-follow） | 纯 AX（覆盖率不稳）、二者结合（v1 过重） |
| 2 | 防自污染 | 私有 pasteboard 类型标记 | 内容指纹（误判漏记）、暂停轮询（漏记窗口） |
| 3 | 读取时序 | 轮询 changeCount 直到变化或超时（500ms 兜底） | 固定 sleep 魔数 |
| 4 | 粘贴后剪贴板归属 | 留下被选条目 X（选取即置顶） | 借用后恢复（反直觉、加竞态） |
| 5 | 多格式存储模型 | 纯文本 + 富文本/HTML 双字段，图片独立 | 单一类型（丢格式）、任意多格式 payload |
| 6 | 敏感跳过 | 平台 concealed 标记 + App 排除名单，不猜内容 | 启发式识别（误判动摇信任） |

### 关键机制细节

- **捕获**：~500ms 轮询 changeCount；捕获纯文本 + 图片 + 富文本/HTML（不含文件路径）。
- **去重**：内容去重 + 置顶刷新——复制已存在内容时不新建，原条目移到最前并刷新时间戳（文本按文本哈希、图片按字节哈希判重）。
- **防自污染**：工具每次自写剪贴板都附私有 UTI 标记（mac 自定义 `NSPasteboard` type、Win 自定义剪贴板格式），轮询见标记即跳过。**"置顶刷新"由业务逻辑显式改库，绝不靠重新捕获。**
- **时序基元**：模拟 Cmd+C 后轮询 changeCount，一递增立即读、500ms 超时兜底（超时判为"无选中"优雅降级）。回写粘贴同理（写完等 changeCount 反映再发 Cmd+V）。
- **回写**：选中条目 → 自动写回剪贴板并模拟粘贴（回车粘贴 / 修饰键仅复制）；粘贴后剪贴板留被选条目 X。
- **隐私三件套**：concealed/transient 平台标记自动跳过（mac `org.nspasteboard.ConcealedType`/`TransientType`、Win `ExcludeClipboardContentFromMonitorProcessing`/`CanIncludeInClipboardHistory=0`）+ App 排除名单 + 托盘暂停开关 + 数据库加密。

> **设计要点**：第 3 题的"等 changeCount 就绪"机制在选中即译（读）和历史回写（写）两头复用，是 v1 操纵剪贴板的统一时序基元。

## 四、翻译子系统

### 4.1 provider 可插拔接口（架构骨架）

核心目标：**加一家新翻译源的边际成本最小**。

| # | 分支 | 定论 | 否决项 |
|---|---|---|---|
| 1 | 契约边界 | 薄 provider + 厚核心框架（横切全下沉） | 厚 provider（各写各的会长歪） |
| 2 | 语言归一 | 本地检测定方向 + 内部 BCP-47 统一 + 每 provider 映射表 | 全靠 provider auto-detect（目标语来不及定） |
| 3 | 错误降级 | 统一错误枚举 + 精准提示 + 同源退避重试，跨源切换永远显式 | 自动 failover 跨源（违背不静默切换） |
| 4 | 凭据配置 | provider 声明 schema + 框架动态渲染 + secret 进 keychain | 各自 UI（违背薄 provider）、通用 KV 框（丢字段语义） |
| 5 | 缓存 | 键含 provider，落库持久 + 容量 LRU 淘汰 | 跨源共享键（劫持换源）、纯内存（重启丢） |
| 6 | 配额 | 被动撞错误 + 本地计数仅展示 | 本地硬追踪拦截（跨设备/时区/计法不可控） |
| + | 注册/超时 | 编译期静态注册表（Rust trait+registry）；超时取消归框架，连续选中即译取消在途旧请求 | — |

**薄 provider 只实现**：① 声明能力（支持语言/是否需 key/配额上限）；② 把内部统一请求翻成自家 HTTP 调用；③ 把响应/错误解析回统一结果 / 统一错误枚举。其余（语言归一、缓存、限流、凭据存取、重试、超时、取消）全由核心框架横切统一。

**错误降级关键区分**：同源瞬时错误（网络抖动 / 429）自动退避重试**同一源**（不算切换）；换到另一个源永远要用户显式（"换源重译"或预配备源链）。

### 4.2 翻译源（v1 内置 4 家）

| 源 | 鉴权 | 角色 | 说明 |
|---|---|---|---|
| **MyMemory** | 匿名 / 邮箱（可选） | **默认源（无 key）** | 匿名 5000、填邮箱 5 万 字符/天，支持 CORS |
| 百度翻译 | appid + 密钥 | 内置 | — |
| DeepL Free | auth key | 内置 | — |
| Google | 官方 API key | 内置 | 走官方 key 版，不用非官方端点 |

Azure Translator / LibreTranslate 自建 / OpenAI 兼容 LLM = 接口留口、**fast-follow 增量加**，不进 v1。失败/额度处理 = 显式提示 + 本地缓存去重；撞额度给明确提示并引导填邮箱或 key；**不静默切换非官方源**。

### 4.3 触发与呈现 UX

- **触发**（已纠偏）：选中即译不再是"选中就自动弹"，而是 **选中 → 光标旁冒不打扰小图标 → 用户主动点击或按 `Cmd+Shift+T` → 才展开轻量浮窗**。避免高频选字时翻译窗乱弹 + 狂烧额度。
- **呈现按触发分**：选中即译 → **光标旁轻量浮窗**（瞥一眼即走、点别处消失）；剪贴板 / 历史条目 / 手动输入 → **固定面板**（原文 + 译文上下对照）。
- **方向**：智能双向——本地检测源语言，自动识别 → 默认中文；本是中文 → 翻英文；目标语可配。
- **译文操作**：一键复制 + 发音朗读（webview 原生 SpeechSynthesis，跨平台免费）+ 切目标语 / 换源重译 + 存入翻译历史。
- **单词查询**：v1 纯机器翻译；词典式增强（音标 / 词性 / 例句）= fast-follow，结果视图预留词典区位置。
- **翻译历史与剪贴板历史分开**：语义不同（翻过什么 vs 复制过什么）。剪贴板历史每条带"一键翻译"按钮 → 翻译并写入翻译历史。

## 五、图片存储与库体积平衡

| # | 分支 | 定论 | 否决项 |
|---|---|---|---|
| 1 | 原图存储 | 原样无损存 + 超阈值才特殊处理 | 全转码压缩（毁保真）、只存缩略图（功能残废） |
| 2 | 超大图处理 | 跳过原图 + 留缩略图 + 明确标记（默认），阈值/压缩可配 | 自动有损压缩（背后改图）、不设阈值 |
| 3 | 缩略图规格 | WebP / 最长边 ~256px（retina 可 320）/ 质量 ~75 | JPEG（透明发黑）、PNG（体积大、不利同步） |
| 4 | 清理策略 | 分级清理：先腾原图保缩略图，再整条清最旧非收藏 | 一刀整条清、仅按张数封顶 |

- **图片入库**：进 SQLCipher 加密库（BLOB），表拆**缩略图 BLOB + 原图 BLOB** 两块；单张超阈值（如 >20MB）默认只留缩略图并标"原图过大未存"。
- **总量封顶**：默认 500MB（或按张数），超出按分级清理——先删大头原图、保留缩略图预览，缩略图也满才整条删最旧非收藏；**收藏永远豁免**。

> **关键架构收敛**：「超大图未存原图」+「本地清理腾掉原图」+「v2 同步原图按需未拉」三种情况最终都归一到**同一记录状态「有缩略图、无本地原图」**——UI、数据模型、"按需重取原图"逻辑只需处理这一种降级态，三处复用一套。

## 六、数据库与加密

| # | 分支 | 定论 | 否决项 |
|---|---|---|---|
| 1 | 失败/恢复语义 | 分级处理 + 永不静默删库 | 失败即重建空库、硬停卡死 |
| 2 | 密钥可访问性 | AfterFirstUnlock + 设备绑定不同步 | WhenUnlocked（锁屏写入空洞）、平台漫游（毁信封模型） |
| 3 | 密钥层抽象 | 抽象 KeyProvider，v1 只实现钥匙串 | v1 即建信封、写死不抽象 |
| + | KDF/性能 | SQLCipher 4 默认参数；搜索靠透明解密后匹配，FTS 留实现期 | — |

- **加密密钥管理 = 钥匙串自动密钥**：随机密钥存 OS 钥匙串（mac Keychain / Win 凭据管理器），日常零摩擦不输密码。安全边界 = 能登入系统者可读（自用够）。
- **密钥可访问性**：`AfterFirstUnlock`（首次解锁后即可读，锁屏后台仍捕获不漏记）+ `ThisDeviceOnly`（绝不同步到 iCloud 钥匙串/凭据漫游）——这是 v2 信封模型的前提（跨设备只能走 DEK 信封端到端）。
- **失败兜底**：瞬时（钥匙串被拒/锁）→ 提示重试 + 引导授权、绝不碰库；永久（密钥丢失/库损坏）→ 明确告知 + **旧库改名备份保留（绝不删）** + 用户显式确认才重建空库。
- **密钥层抽象**：开库只依赖 `KeyProvider` 接口；v1 仅实现 `KeychainKeyProvider`，未来"主口令包裹""v2 共享 DEK"都是新增实现、开库核心不改。
- **备份 / 搬运**：导出生成口令保护的便携文件，在另一台输入口令导入——充当 v1 无同步时的手动搬运通道，也是加密失败时的恢复退路。

> **方法论对照**：密钥层选"只留抽象接口、不预埋实现"，而数据 schema 选"预埋字段"——同一判断的两面：schema 须让数据形状先存在迁移才便宜；密钥来源靠一层薄接口就能彻底解耦。分界 = 这东西事后补的代价高不高。

## 七、跨机同步（v2）

数据层 v1 即预埋，引擎实现推迟到 v2。

| # | 分支 | 定论 | 否决项 |
|---|---|---|---|
| 1 | 同步粒度 | 记录级增量合并（UUID 去重 + 时间戳收敛） | 整库文件搬运（多写冲突/损坏） |
| 2 | 传输架构 | 云盘中转 + 每设备 append-only 加密日志 | P2P（要同时在线）、自建端点（维护重） |
| 3 | 密钥/信任根 | 信封模型：高熵随机 DEK + 一次性配对码包裹 | 长期口令直接派生（泄露=全泄露） |
| 4 | 冲突收敛 | LWW 后写胜 + 墓碑软删 | 完整 CRDT（过度）、仅同步新增（割裂） |
| 5 | 同步范围 | 文本全同步 + 图片缩略图先行/原图按需；配置·密钥不同步 | 全量含原图（云盘爆）、纯文本（丢图） |
| 6 | 触发节奏 | 本地写即时 + 可配间隔轮询 + 手动立即同步按钮 | — |
| 7 | 日志压实 | 全设备最低水位压实 + 墓碑 GC + 离线退役 | 仅体积阈值（漏离线设备增量） |
| 8 | 设备撤销 | 软撤销 + 可选 DEK 轮换（分级） | 仅软撤销（旧设备仍能解密） |

- **每设备 append-only 加密日志**：每台设备只写自己的日志文件（`device-X.log`），别设备只读不改 → 从根上无并发写冲突；数据先 E2E 加密再落盘，云盘商只见密文。默认走 Dropbox/OneDrive（iCloud Drive 因 Windows 缺位不作默认）。
- **信封模型**：首台生成高熵随机 DEK（真正加解密日志的）；配对时用一次性高熵配对码当 KEK 把 DEK 包成 `keybundle` 放云盘，新设备录入配对码解出 DEK 后配对码作废。
- **LWW + 墓碑**：可变字段（收藏/删除/时间戳）各带 last-modified 时间戳，合并取新者；删除写墓碑标记以便传播。
- **设备撤销分级**：「移除设备」轻量出清单（日常换机）；「丢失/被盗」额外触发 DEK 轮换、重加密全部云盘密文使旧设备彻底作废（需至少一台在用设备在线发起）。

## 八、全局热键与弹窗交互

| # | 分支 | 定论 | 否决项 |
|---|---|---|---|
| 1 | 面板定位 | 活动屏固定位（上中，类 Raycast），活动屏取鼠标所在屏 | 鼠标处（键盘唤起乱冒）、记忆位 |
| 2 | 焦点模型 | 抢焦点 + 记住并恢复原前台 app | 非激活面板（搜索框拿不到焦点）、只写回不粘贴 |
| 3 | 窗口生命周期 | 常驻隐藏预热窗口，历史/翻译共用一窗按热键路由 | 按需新建（冷启动卡顿）、超时销毁重建 |
| 4 | 键盘模型 | 搜索框常驻焦点（Spotlight）+ Cmd+1~9 快选 | 列表焦点为主、纯数字快选 |
| 5 | 选中即译触发 | 选中冒图标 + 点按/热键才译 | 纯自动弹（骚扰+烧额度）、修饰键划词 |
| + | 关闭行为 | 失焦即隐 + Esc + 选中后自动关 | — |

- **焦点恢复路径**：唤起前记录前台 app（mac `NSWorkspace.frontmostApplication` / Win foreground window）→ 选中条目时隐面板 → 激活原 app → 等其到前台 → 模拟粘贴。
- **预热窗口**：启动即建好面板 webview、平时隐藏，热键只 show + 重定位 + 聚焦，瞬开无延迟。历史面板与翻译面板共用这一个预热 webview，按 `Cmd+Shift+C` / `Cmd+Shift+T` 路由到不同视图。
- **键盘流**：弹出焦点在搜索框，输入即实时过滤；↑↓ 移动高亮不离搜索框；Enter 粘贴；Cmd+1~9 快选前九条。

## 九、界面 UI 与设计语言

### 9.1 视觉基调与设计语言

整体基调 = **原生克制·随系统明暗·半透明材质**（类 Raycast / Spotlight），在此方向内细化的具体设计语言：

| 维度 | 定论 |
|---|---|
| 强调色策略 | 固定克制品牌主色（非跟随系统） |
| **品牌主色** | **峡湾青蓝 Fjord Teal-Blue `#3A7CA5`** |
| 形态调性 | 中圆角 ~10px + 适中密度 |
| 材质质感 | 弹窗毛玻璃 + 主窗实色 + 细描边/微色差分层 |
| 字体 | 系统原生（mac SF Pro+苹方 / Win Segoe UI+微软雅黑） |
| 图标 | 线性为主（mac SF Symbols / Win Lucide 同风格） |
| 动效 | 克制即时（弹窗淡入+轻微缩放 <150ms 无弹跳） |
| 明暗 | 深浅双主题随系统切换，主色各调适配明度版 |

> 峡湾青蓝与已确立的 Fjord/Nordic 文档审美**同源**——App 与文档整条工具链气质统一。这是"克制但有自己性格"的落点。

### 9.2 面板布局

| 面板 | 布局 |
|---|---|
| 历史面板（热键唤起） | 列表 + 右侧预览双栏（类 Raycast 剪贴板）；★ 置顶收藏，类型筛选放搜索框右侧下拉 |
| 固定翻译面板 | 上下对照（方向+源条 → 原文 → 译文行内复制/朗读 → 底部预留词典区） |
| 选中即译浮窗 | 极简（译文为主 + 复制 + 展开为完整面板）；小图标紧贴选区 |

### 9.3 主窗口

主窗口 = **左侧边栏，一级入口仅三个：剪贴板 / 翻译 / 设置**。翻译历史归「翻译」内部、剪贴板历史归「剪贴板」内部，均为二级。

| 一级页 | 内部布局 |
|---|---|
| 剪贴板 | 复用列表+预览双栏 + 管理操作（多选删除/批量收藏/类型筛选/按时间分组） |
| 翻译 | 工作区主体 + 翻译历史右栏（点项回填工作区），叠主窗左栏即"导航\|工作\|历史"三栏 |
| 设置 | 二级纵向子项栏（通用/热键/翻译源/隐私/存储/关于）+ 右内容（类 macOS 系统设置） |

### v1 为 v2 同步的预埋（关键字段预埋，用户拍板）

v1 现在就要承接的、原本不在 v1 的活：

2. **图片表拆分**：缩略图 BLOB / 原图 BLOB 拆两块存。
3. **每条记录加** `last-modified` 时间戳 + 墓碑标记字段。

**铁律（事后补不了，第一天就要对）**：每条记录 UUID 稳定唯一、永不复用；时间戳存 UTC、单调可靠。

## 十一、贯穿设计原则

这套设计的几乎每一个"否决"，背后都是同一组判断准则：

1. **不静默改 / 不静默删 / 不静默切**——敏感跳过不猜内容、超大图不偷压、换源永远显式、库失败永不静默删。诚实优先于省事。
2. **能统一的横切关注点只做一次**——薄 provider + 厚框架、KeyProvider 抽象、图片三态归一、布局在弹窗与主窗口间复用。
3. **只死守事后补不了的，其余交给迁移 / 接口**——UUID 与时间戳可靠性预埋，密钥来源靠薄接口解耦。分界 = 事后补的代价高不高。
