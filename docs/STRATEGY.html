北极星

North Star

让高频复制 / 翻译的桌面用户——复制即记得住、要翻一键就翻——而工具本身像系统的一部分般即用即走、不打扰、不泄密。

G

目标 Goals

永久稳定 ID，不复用；下游 acceptance.yaml 经 serves_goals 引用它向上锚定。写不出可观察成功信号的目标即太虚、须回炉。

G1

剪贴板"复制必记得住"

剪贴板

复制过的内容总能原样找回并快速取用。

成功信号复制过的文本 / 图片 / 富文本都能在历史里找回；内容去重 + 复制即置顶刷新；纯键盘秒取（输入即搜 / ↑↓ / Cmd+1–9）；回写优先富文本保真、退化纯文本。
G2

翻译"要翻一键就翻"

翻译

在任意取词场景都能低摩擦拿到译文。

成功信号选中→光标旁冒图标 / 按热键即译轻量浮窗；剪贴板 / 历史 / 手输→固定面板原文译文上下对照；多源可插拔，撞额度精准引导（引导填邮箱/key）且永不静默切换非官方源。
G3

"像系统的一部分"的即用即走

体验

随叫随到、看完即走、不喧宾夺主。

成功信号托盘常驻 + 全局热键瞬开（常驻预热窗口零冷启延迟）；原生克制、深浅色随系统；弹窗失焦即隐 / Esc 关 / 选中粘贴后自动关；动效克制即时（<150ms 无弹跳）。
G4

隐私与数据安全是默认

安全

自用工具也不背可被读取 / 泄露的历史。

成功信号SQLCipher 全库加密；concealed/transient 等平台敏感标记自动跳过 + App 排除名单；托盘暂停开关；永不静默删库（密钥丢失/库损坏时旧库改名备份保留、显式确认才重建空库）。
✕

非目标 Non-Goals

明确不做什么——防范围漂移，是 compound engineering 的精华。

  • v1 不做跨机同步——推迟到 v2（数据层 v1 即预埋 UUID + 时间戳 + 可变状态时间戳 + 墓碑，使 v2 为纯新增模块）
  • 不做内容启发式识别密码 / 敏感串（正则/熵猜误判率高，会漏记正常随机串，动摇"复制必记得住"的信任）——只认平台标记 + App 排除名单
  • 不做品牌化重视觉、不做 RTF 富文本捕获/还原（arboard 不支持，成本高收益低）、不做 HTML 内联远程图片下载与完整 CSS 保真
⊟

约束 Constraints

技术 / 资源 / 合规等硬约束。

  • 平台：仅 macOS + Windows；无 App Sandbox；自用、直接分发（非商店）
  • 技术栈固定：Tauri 2.0（Rust）+ React + 嵌入式 SQLCipher（单文件 quickquick.db，引擎静态编入）
  • 密钥设备绑定不同步（标记 ThisDeviceOnly，绝不漫游到 iCloud 钥匙串/凭据漫游）——这是 v2 同步信封模型（DEK 端到端 + 轮换/撤销）成立的前提
  • mac 自动粘贴 / 读取选中文字依赖 Accessibility 授权：未授权须优雅降级不崩（粘贴退化为仅写回剪贴板），不得硬卡用户
▣

如何被消费

本北极星跨版本稳定，不属于版本树。版本验收向下溯源设计文档（source_anchor）的同时，经 acceptance.yaml 头部 serves_goals 向上锚定这里的目标。

  • C1 失锚：版本 serves_goals 空 / 缺 → freeze-lint.sh 阻断冻结
  • C2 悬空锚：引用的目标 ID 不在本文 Goals → 阻断冻结
  • C3 过期锚：版本 strategy_freeze 与本文 STRATEGY@2026-06-07 不一致 → 强制重对齐重冻结
  • 变更纪律：冻结后改本文走 change_log 并更新 strategy_freeze；AI 只起草不发明，作者须是人确认过的产品意图
